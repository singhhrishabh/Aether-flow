WHAT THIS SIMULATES
In Aether-Flow the ESP32 GPIO sends a 25kHz PWM signal through an IRF540N MOSFET to control each TEC1-12706 Peltier module. This circuit uses a 2N2222 transistor instead — same concept, safer voltage for Tinkercad simulation. Turn the potentiometer and watch motor speed change — that is exactly how Peltier cooling intensity is controlled.

WHAT THIS SIMULATES
This is the exact PID algorithm running on the ESP32 as Aether-Flow's safety fallback. Same Kp, Ki, Kd values. If the AI goes silent for 2 seconds, this loop takes over automatically and keeps the chip cool. The LCD shows live temperature, setpoint, PWM output, and error — identical to what the ESP32 Serial Monitor displays.

HOW TO TEST IN TINKERCAD
1. Run simulation
2. Green LED glows — system normal
3. Drag TMP36 slider above 45°C
4. Red LED fires, relay opens, motor stops
5. Drag back below 45°C — system resets

In real Aether-Flow: threshold is 85°C and relay cuts all Peltier power in under 10ms — software-independent, always active.
